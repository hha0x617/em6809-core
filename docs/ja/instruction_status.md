# 命令カバレッジと表記・検証（MC6809・日本語）

本書は em6809-core の命令カバレッジ、逆アセンブルの表記規約、フラグ/副作用の方針、実用的な検証手順を日本語でまとめたものです。本ファイルは `docs/en/instruction_status.md` (canonical) の日本語版です。

> **注:** 検証例で参照しているサンプルプログラムや CLI ランナーは
> [em6809](https://github.com/hha0x617/em6809) (本クレートをラップする
> GUI ホスト) に同梱されています。em6809-core 自体はライブラリ専用で、
> em6809 / [emfe MC6809 プラグイン](https://github.com/hha0x617/emfe_plugins/tree/master/mc6809)
> / 独自の embedder と組み合わせて、本書中の trace サンプルを実行
> してください。

## ステータス (2026-04-19)

**標準 MC6809 ISA は 100% 実装済み**。メインテーブル 256 opcode 空間のうち
223 個の正規命令すべてと `$10`/`$11` プレフィクスのサブ命令一式をデコード・
実行できます。残り 33 個は ISA 上の reserved / illegal slot のため、意図的に
no-op フォールバックとしています。

2026-04-19 に長年残っていた gap と 2 件の正確性バグ (LEAS/LEAU 入替、
Indexed の PC-relative 誤判定) を解消しました。詳細は末尾の
[実装履歴](#実装履歴) を参照してください。

## 概要

- 本エミュレータは MC6809 の**全命令と全アドレッシングモード**を実装。短/長
  分岐、A/B/D 演算、X/Y/U/S ロード/ストア、メモリ シフト/ローテート/クリア/
  テスト/ジャンプ、スタック操作、制御命令 (MUL, SEX, DAA, SYNC, CWAI,
  SWI/SWI2/SWI3, ABX 等)。
- 逆アセンブラはページ 0x00/0x10/0x11 を網羅し、被演算子（即値/ダイレクト/インデックス/拡張）と間接表記、分岐オフセット→ターゲットの可視化を統一形式で出力。

## 逆アセンブルの表記仕様（抜粋）

- 即値: `#${nn}`（例: `#${10}`）
- ダイレクト/拡張: `${addr}`（例: `${E000}`）
- インデックス: `±off,REG`（例: `-4,X` / `8,U`）
- 間接: 上記被演算子を角括弧で包む（例: `[10,X]`、`[${E040}]`）
- 分岐: `Bcc/LBcc offset -> $${target}` として相対→絶対を併記

## 命令カバレッジ（サマリ）

- 分岐
  - 短分岐 `Bcc` 全種（`BRA`, `BRN`, `BHI`, `BLS`, `BCC`, `BCS`, `BNE`, `BEQ`, `BVC`, `BVS`, `BPL`, `BMI`）
  - 長分岐 `LBcc` 全種（ページ 0x10/0x11）
  - `BSR`/`LBSR`
- 比較・ロード/ストア
  - `CMPA/B`, `SUBA/B`, `ADCA/B`, `ADDA/B`, `SBCA/B`, `ANDA/B`, `ORA/B`, `EORA/B`, `BITA/B`
  - `LDA/B/D`, `STA/B/D`, `LDX/U`, `STX/U`, `LDY/S`, `STY/S`, `CPX`, `CPY`, `CMPU`, `CMPS`, `LDS`, `STS`
  - アドレッシング: 即値/ダイレクト/インデックス/拡張（各命令の規定モードに準拠）
- メモリ操作・シフト/ローテート/テスト
  - `ASL/ASR/LSL/LSR`, `ROL/ROR`, `TST`, `CLR`, `COM`
  - 対象: ダイレクト/インデックス/拡張、累積器版（A/B）該当種
- 転送・合成・制御
  - `JMP`（D/X/Y/U/S/拡張/インデックス/間接対応），`JSR`（各モード）
  - `LEAX/Y/U/S`, `PSHS/U`, `PULS/U`, `EXG`, `TFR`
  - `SYNC`, `CWAI`, `MUL`, `DAA`, `SEX`, `ABX`, `ANDCC`, `ORCC`, `SWI/SWI2/SWI3`

## カバレッジ行列表（運用確認用）

|---|---|:--:|:--:|:--:|:--:|
| 8-bit 演算A/B | `ADDA/B`, `ADCA/B`, `SUBA/B`, `SBCA/B` | ✓ | ✓ | ✓ | ✓ |
| 論理/ビット | `ANDA/B`, `ORA/B`, `EORA/B`, `BITA/B` | ✓ | ✓ | ✓ | ✓ |
| 比較A/B | `CMPA/B` | ✓ | ✓ | ✓ | ✓ |
| ロード | `LDA/B`, `LDD` | ✓ | ✓ | ✓ | ✓ |
| ストア | `STA/B`, `STD` | — | ✓ | ✓ | ✓ |
| インデクス | `LDX/U`, `STX/U` | ✓/— | ✓ | ✓ | ✓ |
| インデクス | `LDY/S`, `STY/S` | ✓/— | ✓ | ✓ | ✓ |

| スタック | `LDS`, `STS` | ✓/— | ✓ | ✓ | ✓ |

|---|---|:--:|:--:|:--:|
| メモリ操作 | `TST`, `CLR`, `COM` | ✓ | ✓ | ✓ |

| 項目 | 内容 | 状態 |
|---|---|---|
| 短分岐 | `Bcc` 全種 | ✓ |

注記

- `Imm` は即値、`Indexed` は全バリアント（5-bit/8-bit/16-bit 変位、間接を含む）を網羅する方針。
- `LDX/U` と `LDY/S` の Imm 列は、対応命令のみ ✓（一部は Immediate を持たないため `✓/—` と表記）。
- サイクル数は命令主体で整備済み。メモリ待ちなどプラットフォーム依存の遅延は未モデル化。

## フラグと副作用（方針）

- N/Z/V/C は公式 MC6809 の定義に準拠。`DAA/MUL/CWAI` も仕様に従う。
- 比較系（`CMP*`, `CP*`）は減算相当のフラグ生成でデータは不変。
- `ANDCC/ORCC` は CCR をビットマスクで更新。`CWAI` は CCR と待機状態を処理。

## 既知の仕様/表示

- 分岐は「相対オフセット」と「絶対アドレス」を併記。トレースでは解決後の PC を表示。
- インデックス間接は角括弧表記。`D` を使う変位など複合形式も統一。

## 検証手順（実用）

- `samples/traces/trace_all.s19`（および `trace_all_startfirst.s19`）を GUI でロードし、トレースでマーカー/フラグ/分岐を確認。
- 実行開始位置の優先順位:
  - `--pc` > リセットベクタ > S‑Record のエントリ（ベクタが無効な場合）> 最初のロードレンジ先頭

本ページは運用に合わせて随時更新します。不足や誤りがあれば Issue へご連絡ください。

## 実装履歴

### 2026-04-19 (後) — Indexed PC-relative 判定バグ修正

`ea_indexed` の extended postbyte (bit7=1) 処理で、PC-relative 判定が誤って
いた:

```rust
// 誤: rsel==3 (S) かつ form=0x04/0x08/0x09/0x0B なら PC-rel として扱う
let is_pc_rel = rsel == 3 && matches!(form, 0x04 | 0x08 | 0x09 | 0x0B);
```

MC6809 では PC-relative は postbyte form=0x0C (8-bit) / 0x0D (16-bit) のみで、
rsel は don't-care。修正後:

```rust
let is_pc_rel = matches!(form, 0x0C | 0x0D);
```

加えて form 0x0C / 0x0D 用のアーム (自前で PC を base に計算) を追加。

症状: `lda ,s` (lwasm は offset=0 のとき postbyte $E4 を生成) が S ではなく
PC 直後の命令バイトを読み、Tiny Lisp Phase 3 の算術プリミティブ (`+`/`-`/`<`)
で `addd ,s` / `cmpx ,s` パターンが壊れていた。

### 2026-04-19 — ISA カバレッジ完了

Tiny Forth (`emfe_plugins/mc6809/examples/forth/forth.asm`) 実装中に残存
gap と 1 件の入替バグが判明したため、まとめて解消。

**重大バグ修正**: `LEAS` ($32) と `LEAU` ($33) のハンドラが入れ替わって
いた — $32 の分岐が `U` を更新し、$33 の分岐が `S` を更新していた（ISA
では逆）。加えて両者が CC を更新していたが、ISA 上 `LEAS`/`LEAU` は CC
を変更しない (Z を更新するのは `LEAX`/`LEAY` のみ)。症状: `leau 2,u` が
リターンスタックポインタ `S` を壊し、以降の `lbsr/rts` で未定義動作。
Tiny Forth では `10 3 - .` が `1927  ok` (= nfa_INTERP のアドレス 0x0787)
を返すバグとして表面化。

**追加命令 (全 33 個)**

- 累算器: `NEGA` ($40), `NEGB` ($50), `ASRA` ($47), `ASRB` ($57)
- 算術: `SEX` ($1D), `MUL` ($3D), `DAA` ($19)
- `ABX` ($3A)
- メモリ Read-Modify-Write（各命令 dir / idx / ext 3 形式）:
  - `NEG` ($00 / $60 / $70)
  - `COM` ($03 / $63 / $73)
  - `LSR` ($04 / $64 / $74)
  - `ROR` ($06 / $66 / $76)
  - `ASR` ($07 / $67 / $77)
  - `ASL/LSL` ($08 / $68 / $78)
  - `ROL` ($09 / $69 / $79)
  - `DEC` ($0A / $6A / $7A)
  - `INC` ($0C / $6C / $7C)
  - `TST` ($0D / $6D / $7D)
- 割込み・同期: `SYNC` ($13), `CWAI` ($3C)
  - どちらも「割込み待機」動作は no-op として近似 (フォールスルー)。
    `CWAI` のスタックフレーム push は仕様通り実行。

### 残存 gap (reserved / illegal — 実装不要)

下記 33 スロットは実際の MC6809 でも**未定義または不正命令**です（`$87 STA #`
のように意味を持たない immediate-store 系や、Motorola 未割当スロット）。
意図的に未実装のままにしています:

```
$01 $02 $05 $0B $14 $15 $18 $1B $38 $3E
$41 $42 $45 $4B $4E
$51 $52 $55 $5B $5E
$61 $62 $65 $6B
$71 $72 $75 $7B
$87 $8F  $C7 $CD $CF
```
